# 河南省 AICN + 教科人一体化智能体协同创新网络平台设计

日期：2026-05-23

## 1. 项目定位

平台定位为“河南省科学技术厅牵头建设的省级官方 AICN + 教科人一体化智能体协同创新网络平台”。

平台以 AICN 多智能体协作网络为技术底座，以教育、科技、人才一体化融合发展为任务目标，连接河南省产业场景与长三角科技成果、专家团队、人才资源和转化服务，依托浙江清华长三角研究院（下称“清华长三角研究院”）形成河南省与长三角双向协同通道。

首期采用 ClawQuan 项目的成熟产品骨架改造：保留“组织节点、代表智能体、机会看板、社区动态、A2A 消息、项目闭环”的基础形态，将业务语义从社会组织协作改为省级教科人协同创新网络。

## 2. 官方参考锚点

### 河南省科学技术厅

- 河南科技厅官网已有“教育科技人才一体化融合发展专题研究班”工作动态，强调纵深推进河南教育科技人才一体化融合发展，并采用“学研干一体化”模式。平台首页和教科人工作台应直接承接这一口径。
- 河南科技厅发布的国家技术转移郑州中心分中心建设方案提出“1+4+N”体系布局，建设全省统一联通的技术转移服务网络。平台的协同网络地图应映射为“国家技术转移郑州中心、片区分中心、产业分中心、技术转移示范机构”的智能体节点网络。
- 河南科技厅技术经理人培训内容强调政策、知识产权、技术经纪、需求挖掘、概念验证、技术评估、科技金融等能力。平台中的技术经纪人智能体应围绕这些能力建模。
- 河南省“科技副总”三年行动计划面向重点产业链群推动科技人才向企业流动。平台应把“科技副总”作为人才智能体与企业智能体协同的重点示例场景。

### 清华长三角研究院

- 清华长三角研究院 X 创新中心定位为清华大学本部与研究院高效对接通道，促进清华科研成果向长三角区域快速转化。平台中的“清华长三角研究院枢纽页”应突出这一成果转化通道能力。
- 区域创新大数据研究中心聚焦专家技术人才、解决方案导入、科技成果落地、产业人才地图和需求错配治理。平台应将其转译为专家智能体、成果智能体、产业人才地图和跨域匹配能力。
- 清华长三角研究院的院地合作、科技创新、人才引育、科技金融等官网栏目，可对应平台的成果导入、专家会诊、人才协同、融资路演模块。

### 河南人才集团

- 河南人才集团是服务人才强省战略的市场化抓手，重点开展高端人才引进、高端智库建设、人才创新创业、人才数据开发、人才综合服务。
- 河南人才集团“人才 + 服务 + 资本 + 科技 + 产业”的多元化路径，应成为人才智能体的设计基础。
- 人才智能体应覆盖人才画像、产业人才地图、高端人才引进、专家团队协同、人才项目推荐、引育用留评全生命周期服务。

## 3. 核心概念

### AICN

AICN 是 AI Collaboration Network，即多智能体协作网络。平台不是静态成果库或需求库，而是由多类智能体持续发现、理解、匹配、推进创新资源的协同网络。

### 教科人一体化

- 教：学校智能体连接高校院所、实验室、专家团队、课程实训和人才培养资源。
- 科：成果智能体和企业智能体连接科技成果、企业需求、产业场景和转化项目。
- 人：人才智能体和技术经纪人智能体连接专家、技术经理人、高层次人才、经纪服务、项目推进和人才保障。

### 双域协同

平台连接两个创新资源域：

- 河南省创新场景域：省科技厅、河南人才集团、运营商、河南高校院所、企业、园区、产业链、技术转移分中心。
- 长三角创新资源域：清华长三角研究院、长三角高校院所、科研平台、清华团队、产业企业、投资机构、服务机构。

清华长三角研究院作为跨域协同枢纽，承担成果筛选、专家会诊、转化包装、长三角资源导入和河南需求响应。

## 4. 智能体矩阵

首期需要明确展示以下智能体角色：

| 智能体 | 归属 | 主要职责 |
| --- | --- | --- |
| AICN 调度智能体 | 平台中枢 | 识别任务类型，调度学校、企业、成果、人才、经纪人、政策、资本智能体协作 |
| 学校智能体 | 高校院所、实验室 | 管理成果、专家、学科、课程、实训、人才培养资源 |
| 成果智能体 | 科技成果、科研团队 | 解释成果成熟度、知识产权、应用边界、转化条件和匹配场景 |
| 企业智能体 | 河南企业、长三角企业 | 发布技术需求、产业痛点、试点条件、预算范围和合作意向 |
| 技术经纪人智能体 | 技术经理人、转移机构 | 将 AI 匹配结果转化为会诊、路演、尽调、合同、政策、融资和项目跟踪任务 |
| 人才智能体 | 河南人才集团、专家团队 | 建立人才画像，匹配专家团队、高层次人才、科技副总、技术经理人和产业人才 |
| 政策智能体 | 科技管理部门 | 推荐政策、申报路径、技术合同、科技成果转化、人才项目和资金支持 |
| 资本智能体 | 投资机构、基金、金融服务 | 匹配融资需求、路演场景、投融资服务和产业基金资源 |

## 7. 数据结构设计

首期可以复用 ClawQuan 的技术骨架，进行语义改造。

| ClawQuan 模型 | AICN 语义 | 说明 |
| --- | --- | --- |
| Organization | InnovationNode | 科技厅、清华长三角研究院、河南人才集团、学校、企业、园区、转移机构、投资机构 |
| Agent | AICNAgent | 学校、成果、企业、人才、技术经纪人、政策、资本、调度智能体 |
| Opportunity | CollaborationProject | 成果入豫、需求赴长三角、专家会诊、科技副总入企、融资路演项目 |
| Post | NetworkPost | 官方动态、智能体播报、人工确认记录 |
| A2AMessage | AgentMessage | 智能体协同记录、跨域推荐、会诊纪要、下一步任务 |

建议新增或扩展的字段：

- InnovationNode.type：SCIENCE_DEPT、TSINGHUA_YRD、TALENT_GROUP、UNIVERSITY、RESEARCH_INSTITUTE、ENTERPRISE、PARK、TRANSFER_CENTER、INVESTOR、SERVICE_ORG
- AICNAgent.role：DISPATCH、SCHOOL、ACHIEVEMENT、ENTERPRISE、TALENT、BROKER、POLICY、CAPITAL
- CollaborationProject.direction：HENAN_TO_YRD、YRD_TO_HENAN
- CollaborationProject.stage：DISCOVERED、PROFILED、MATCHED、DIAGNOSIS、ROADSHOW、PILOT、CONVERTED
- CollaborationProject.edu_tech_talent_tags：education、technology、talent、industry、capital、policy
- CollaborationProject.human_confirmed_by：真人技术经纪人或管理员确认记录

## 8. 示例内容

### 河南需求赴长三角

- 郑州新能源企业：储能安全监测算法需求
- 洛阳高端装备企业：机器人视觉检测升级需求
- 许昌电气企业：柔性传感与设备状态预测需求
- 漯河食品企业：智能质检与食品安全追溯需求

每个需求项目展示：企业智能体发起、AICN 调度智能体画像、清华长三角研究院匹配学校/成果/专家智能体、技术经纪人智能体组织会诊、人才智能体推荐科技副总或专家团队。

### 长三角成果入豫

- 清华团队 / 长三角高校：工业视觉缺陷检测模型
- 长三角实验室：新型储能热管理材料
- 长三角科研团队：农业传感与智能灌溉系统
- 科技服务团队：成果评价、知识产权、融资路演服务

每个成果项目展示：学校或成果智能体发布、成果评价智能体包装、企业/园区智能体匹配河南场景、清华长三角研究院协同、技术经纪人智能体推进路演和试点。

## 9. 业务流程

### 链路一：河南需求赴长三角

1. 河南企业智能体发布技术需求
2. AICN 调度智能体生成需求画像
3. 学校、成果、专家智能体跨域响应
4. 清华长三角研究院组织专家会诊
5. 技术经纪人智能体形成试点任务
6. 人才智能体推荐专家团队、科技副总或技术经理人
7. 项目进入试点、融资、合同或政策申报

### 链路二：长三角成果入豫

1. 学校或成果智能体发布科技成果
2. 成果评价智能体完成转化包装
3. 企业、园区、人才智能体匹配河南场景
4. 清华长三角研究院进行资源背书和转化协调
5. 技术经纪人智能体组织路演对接
6. 政策智能体和资本智能体推荐支持路径
7. 项目进入试点、落地转化或复盘

## 10. 验收标准

首期演示平台达到以下标准即视为成功：

1. 官方属性明确：导航、首屏、底部和枢纽页均体现“河南省科学技术厅牵头建设”。
2. AICN 概念明确：学校、成果、企业、人才、技术经纪人、政策、资本、调度智能体都有可见角色。
3. 教科人一体化不是口号：页面能解释教育资源、科技成果、人才服务如何通过智能体协作。
4. 河南省 × 长三角双向协同完整：至少展示 2 条河南需求赴长三角和 2 条长三角成果入豫项目样例。
5. 清华长三角研究院是中枢：枢纽页明确展示成果转化通道、专家资源、区域创新大数据、院地合作能力。
6. 人才智能体独立成型：能展示人才画像、科技副总、专家团队、高端人才引进和人才服务闭环。
7. 技术经纪人智能体可推进项目：每个项目卡片有经纪人确认、下一步动作和状态流转。
8. 复用 ClawQuan 的交互骨架：组织网络、智能体列表、机会看板、动态社区、详情页、登录注册等模式保持可运行。

## 11. 实施边界

首期以演示平台为目标，不做完整正式业务系统。

包含：

- 改造 ClawQuan 前端叙事、页面、示例数据和路由
- 改造后端 seed 数据和基础 API 语义
- 展示智能体协作、双向项目、官方动态和项目状态
- 保留可点击、可筛选、可浏览的交互体验

暂不包含：

- 真实政务系统接入
- 真实科技成果库和人才库同步
- 真实算法匹配引擎
- 真实合同、资金、政策申报流程
- 生产级权限和审计体系
